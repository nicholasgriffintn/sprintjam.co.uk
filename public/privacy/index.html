How SprintJam handles your data

SprintJam is built to stay lightweight: no ads, no marketing trackers, and minimal storage. Everything below is what the app actually keeps and the configuration needed to keep it private when you self-host.

What lives in your browser

  • Your display name and theme preference are saved to localStorage so you do not have to re-enter them.
  • No cookies or cross-site tracking pixels are used.

What the server keeps

  • Room state stays inside a Cloudflare Durable Object bound to your account: room key, moderator, hashed passcode, participant names/avatars, connection status, votes (including structured votes), and ticket queue metadata.
  • Optional room music stores the generated Strudel code and playback state alongside the room.
  • Data is not shared with third-party analytics; traffic stays within your Cloudflare zone unless you enable an integration.

Optional integrations

  • Jira (optional): Only used if you set JIRA_OAUTH_CLIENT_ID, JIRA_OAUTH_CLIENT_SECRET, and JIRA_OAUTH_REDIRECT_URI. When enabled, per-room OAuth tokens and ticket metadata are stored inside the Durable Object.
  • Strudel music (optional): Requires POLYCHAT_API_TOKEN. When set, we send a short prompt (no room names or user identifiers) to api.polychat.app to generate code snippets.

Privacy configuration required for self-hosting

  • Logging/observability: by default invocation logs are off, but request sampling is on. Set sampling to 0 (or disable the block) if you need zero log retention in Cloudflare: 
{
  "observability": {
    "enabled": true,
    "head_sampling_rate": 0,
    "logs": { "enabled": false, "head_sampling_rate": 0 }
  }
}
Adjust this in wrangler.jsonc before deploying to match your policy.
  • Serve the site over HTTPS with your own Cloudflare domain and keep any WAF/Zero Trust rules aligned with your team's privacy requirements.
